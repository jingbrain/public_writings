操作流程总结

# 第一阶段：搜集素材

1. Gemini 3提示词
  >搜索并总结 [主题] 的最新 10 个可靠来源，提取关键事实、数据、争议点，按时间/观点分类，并把结果以markdown输出到文件 '**素材_Gemini.md**'。


2. Grok 4.2 Beta 提示词
  >基于保存在文件 '**素材_Gemini.md**'里得素材，用实时 X 和网页验证最新动态，补充任何 2026 年新进展或反面，包括【新增】、【已验证】、【更新】、【标记】, 并把结果以markdown输出到文件 '**素材_Grok.md**' 

3. Claude Sonnet 4.5 提示词
  >你现在收到两份关于【主题】的素材，分别存在以下两个文件：
     - A: **素材_Gemini.md**
     - B: **素材_Grok.md**
  >任务：把 A 和 B 合并成一份最终干净素材包。
  要求：
     1. 保留 Gemini 原有的主要分类结构（时间线 + 观点分类），作为主骨架。
     2. 把 Grok 的所有【新增】内容插入到最匹配的分类里。
     3. 对 Grok 的【已验证/更新】部分，优先采用最新表述，删除旧版重复。
     4. 任何冲突或矛盾，以 Grok 的实时版本为准，但如果 Grok 删除了 A 中的内容，要在对应位置加一句【历史背景：原数据/观点为……，现已更新/证伪】。
     5. 去掉所有重复表述，精简语言，但保留关键数据、来源链接、数字。
     6. 最后加一个小节“素材最终总结”：列出本次合并后新增/更新的关键点（3-8条）。
     7. 输出格式保持清晰、可读，用 markdown 或纯文本分级标题,保存到文件 '**素材_sonnet.md**'
     8. 输出只包含最终合并后的完整素材包，不要加额外解释。

# 第二阶段：文章骨架构思

Claude Opus 4.5提示词：
  >基于文件 '**素材_sonnet.md**' 里的素材，为一篇 [字数/类型] 文章构建完整骨架：
  >1. 核心论点 
  >2. 分节逻辑（每节 3-5 个子点 + 过渡理由）
  >3. 潜在反驳与回应 
  >4. 建议开头/结尾钩子。要求严谨、原创、平衡。
  >5. 以markdown格式输出大纲文件 '**大纲.md**'

# 第三阶段：文章成稿

## 写初稿
Sonnet 4.5 提示词
  >请根据素材 '素材_sonnet.md'，严格按照大纲 '大纲.md'写完整初稿，要求语言流畅、专业、适合 [目标读者]，以 markdown 格式输出到 '**输出1_初稿.md**'。

## 精炼
Opus 4.5 提示词
  >润色初稿 '输出1_初稿.md'，提升连贯性、删减冗余、加强论证力度和文采.
  >以 markdown 格式输出到文件 '输出2_精炼.md'。

## 创意润色
GPT 5.2 提示词
>基于 ‘输出2_精炼.md’，在保持原意前提下，让语言更生动、节奏更好、增加钩子
>以 markdown 格式输出到文件 '输出3_润色.md'。

# 终审
Grok 4.2 Beta 提示词
>对 '输出3_润色.md' 事实核查，并提出任何遗漏视角或风险
>以 markdown 格式输出到文件 '输出4_润终稿.md'。
